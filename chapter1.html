Database Concepts

Data

Data refers to raw facts and figures that have no context or meaning on their own. For example, numbers, names, dates, etc.

Information

Information is processed data that has meaning and is useful for decision-making. For example, a report showing sales over time.

Features of Information

  • Accuracy: Free from errors.
  • Relevance: Related to the subject.
  • Completeness: All necessary data is included.
  • Timeliness: Available when needed.
  • Consistency: Same across all systems.

Database and Its Purpose

A database is an organized collection of data that can be easily accessed, managed, and updated. The purpose is to store, retrieve, and manage data efficiently.

Terminologies in Database

  • Table: A collection of related data organized in rows and columns.
  • Field: A column in a table representing a single attribute of data.
  • Record: A row in a table; a complete set of fields.
  • Tuple: Another term for a record.
  • Object: A real-world entity in object-oriented databases.
  • Keys: Identifiers used to uniquely identify records (e.g., Primary Key, Foreign Key).

Data Dictionary

A data dictionary is a centralized repository of information about data such as meaning, relationships to other data, origin, usage, and format.

Database Management System (DBMS)

Introduction

A DBMS is software used to create, manage, and manipulate databases. It allows users and applications to interact with data easily.

Objectives

  • Efficient data management
  • Data security and integrity
  • Data sharing among multiple users
  • Reduce data redundancy

Advantages

  • Improved data sharing
  • Data security and access control
  • Minimized data inconsistency
  • Backup and recovery options

Disadvantages

  • Complexity in setup and management
  • High cost of hardware and software
  • Requires skilled personnel

Types of Database Models and Integrity Constraints

Types of Database Models

  • Hierarchical Model: Organizes data in a tree-like structure with parent-child relationships.
  • Network Model: Allows many-to-many relationships using a graph-like structure.
  • Relational Model: Represents data in tables with rows and columns.
  • Entity-Relationship (ER) Model: Uses entities, attributes, and relationships to represent data conceptually.

Integrity Constraints and Their Types

Integrity constraints ensure the accuracy and consistency of data in a database.

  • Domain Constraint: Ensures correct data type and format.
  • Entity Integrity: Primary keys must be unique and not null.
  • Referential Integrity: Foreign keys must match existing primary keys or be null.
  • Key Constraints: Ensure uniqueness and identification.

Normalization

Introduction

Normalization organizes data to reduce redundancy and improve integrity.

Normal Forms

  • 1NF: Values in each column are atomic and one value per row.
  • 2NF: Table is in 1NF and all non-key attributes fully depend on the primary key.
  • 3NF: Table is in 2NF and all attributes only depend on the primary key.

Advantages

  • Reduces redundancy
  • Improves consistency
  • Better maintenance

Disadvantages

  • Complex queries
  • More joins
  • Performance may reduce

Centralized & Distributed Databases, Database Security, and SQL Basics

Centralized and Distributed Database

Introduction

A Centralized Database is managed in one location.
A Distributed Database is spread over multiple locations.

Advantages

  • Centralized: Easier to manage and secure.
  • Distributed: Faster local access and fault-tolerant.

Disadvantages

  • Centralized: Single point of failure.
  • Distributed: Complex and harder to secure.

Comparison

Feature Centralized Distributed
Location Single site Multiple sites
Access Speed Slower for remote users Faster for local users
Complexity Simple Complex
Reliability Lower Higher
