Database Management System (DBMS) and C Programming – Questions and Answers

Database Management System (DBMS)

1. What is Database and DBMS? List out the advantages and disadvantages of DBMS.

Database: A database is a structured collection of related data that can be easily accessed, managed, and updated.

DBMS (Database Management System): It is software used to store, retrieve, and manage data in databases. Examples include MySQL, Oracle, and PostgreSQL.

Advantages of DBMS:

  • Reduces data redundancy
  • Ensures data consistency
  • Improves data security
  • Provides backup and recovery
  • Supports multi-user environment

Disadvantages of DBMS:

  • High cost of hardware and software
  • Requires trained personnel
  • Complex to design and maintain
  • System failure can lead to data loss

2. Differentiate between file processing system and DBMS (at least 4 points).

  • Data Redundancy: File system has more redundancy; DBMS reduces redundancy.
  • Data Consistency: File system may cause inconsistency; DBMS ensures consistency.
  • Security: File system has poor security; DBMS provides high-level security.
  • Access: In file system, accessing data is complex; in DBMS, it is easier with SQL.

3. Explain the different models of DBMS with advantages and disadvantages.

1. Hierarchical Model: Organizes data in a tree-like structure.

  • Advantages: Simple to design, fast access for hierarchical relationships.
  • Disadvantages: Rigid structure, difficult to modify relationships.

2. Network Model: Data is represented by records connected with links.

  • Advantages: More flexible than hierarchical model, handles many-to-many relationships.
  • Disadvantages: Complex structure, hard to maintain.

3. Relational Model: Data is stored in tables (relations).

  • Advantages: Easy to use, powerful query language (SQL), flexible.
  • Disadvantages: Slower for complex relationships.

4. What is a relational database? How is it different from other database models?

A relational database stores data in the form of tables (rows and columns) with relationships among the tables.

Differences:

  • Uses tables instead of trees or networks.
  • Supports SQL for data operations.
  • Provides easier data manipulation and retrieval.
  • More suitable for modern applications.

5. What is data redundancy? How does DBMS help in reducing it?

Data redundancy: It is the unnecessary repetition of data within a database or storage system.

DBMS reduces redundancy by storing data centrally and using relational techniques like normalization to remove repeated data across multiple tables.

6. Differentiate between centralized and distributed database systems.

  • Centralized: All data is stored in a single location. Easier to manage but vulnerable to failures.
  • Distributed: Data is stored across multiple locations. Offers better availability and speed but is more complex to manage.
  • Centralized requires low maintenance cost, distributed requires higher network coordination.
  • Distributed databases are more scalable compared to centralized ones.

7. Who is Database Administrator (DBA)? What are the major responsibilities of DBA?

Database Administrator (DBA): A person responsible for managing and overseeing database systems.

Responsibilities:

  • Database design and implementation
  • Data security and backup
  • Performance tuning
  • User access management
  • Recovery from failures

8. Define normalization. Explain 1NF, 2NF and 3NF with suitable examples.

Normalization: It is the process of organizing data to minimize redundancy and improve data integrity.

1NF (First Normal Form): Ensures that each column contains atomic values and no repeating groups.

Example: A table with multiple phone numbers in one column is not in 1NF. Splitting them into separate rows makes it 1NF.

2NF (Second Normal Form): It is in 1NF and all non-key attributes are fully functionally dependent on the primary key.

Example: If a student table has course name depending on course ID, it should be in a separate table.

3NF (Third Normal Form): It is in 2NF and there are no transitive dependencies.

Example: If a student table has 'Department Name' depending on 'Department ID', move department data to a new table.

9. Explain the terms: primary key, foreign key, and candidate key with examples.

  • Primary Key: A column (or set of columns) that uniquely identifies each record in a table.
    Example: StudentID in the Student table.
  • Foreign Key: A column that establishes a relationship between two tables.
    Example: DepartmentID in Student table referencing Department table.
  • Candidate Key: All possible columns that can be a primary key.
    Example: In Employee table, both Email and EmployeeID can be candidate keys.

10. What is SQL? Explain its components and common functions.

SQL (Structured Query Language): It is used to manage and manipulate relational databases.

Components of SQL:

  • DDL (Data Definition Language): CREATE, ALTER, DROP
  • DML (Data Manipulation Language): SELECT, INSERT, UPDATE, DELETE
  • DCL (Data Control Language): GRANT, REVOKE
  • TCL (Transaction Control Language): COMMIT, ROLLBACK

Common Functions:

  • COUNT() – counts rows
  • SUM() – adds values
  • AVG() – finds average
  • MAX(), MIN() – find highest/lowest values

12. Define the following terms.

a. Data Dictionary:
A data dictionary is a centralized repository that stores definitions, relationships, and formats of data elements in a database. It helps in managing metadata (data about data).

b. Primary Key:
A primary key is a column or set of columns in a table that uniquely identifies each record. It cannot be null and must contain unique values.

c. Relationship:
A relationship in a database refers to the association between tables based on common fields, usually implemented using primary and foreign keys.

d. Data Manipulation Language (DML):
DML is a subset of SQL used to retrieve and manipulate data in a database. Common DML commands include SELECT, INSERT, UPDATE, and DELETE.

e. Structured Query Language (SQL):
SQL is a standard programming language used to manage and operate relational databases. It allows creating, modifying, and querying database records.

f. Data Integrity:
Data integrity ensures the accuracy and consistency of data over its entire lifecycle. It is maintained using constraints like primary keys, foreign keys, and unique constraints.

g. Data Definition Language (DDL):
DDL is a subset of SQL used to define the structure of database objects such as tables, indexes, and views. Examples: CREATE, ALTER, DROP.

h. Data Security:
Data security refers to the protection of data from unauthorized access, corruption, or loss. DBMS provides access control, encryption, and authentication for security.

i. Database System:
A database system is a combination of the database, the DBMS software, and the applications that interact with them. It provides tools to create, store, modify, and retrieve data efficiently.

C Programming

1. Working with a Function

a. Define Function:
A function is a block of reusable code designed to perform a specific task. It helps in reducing repetition, increasing modularity, and improving code clarity in programming.

b. Syntax of Functions:

return_type function_name(parameters) {
    // body of the function
    return value;
}

c. Types of Functions:

  • Library Functions: Predefined functions provided by programming languages (e.g., printf(), sqrt()).
  • User-defined Functions: Functions created by the programmer to perform specific tasks.

d. Components of a Function:

  • i. Function Prototype:
    It declares the function to the compiler before its actual definition. Example:
    int add(int, int);
  • ii. Function Call:
    This is where the function is actually used. Example:
    add(5, 3);
  • iii. Function Definition:
    This contains the actual code of the function. Example:
    int add(int a, int b) {
    return a + b;
}
  • iv. Return Type:
    Specifies the type of value a function returns (e.g., int, float, void). For example:
    int sum(int a, int b); means the function returns an integer.

2. Categories of Function with Examples

Functions can be categorized based on whether they take arguments and whether they return values.

  • i. Function with return type but no arguments:
    This function returns a value but takes no parameters. 
    int getNumber() {
    int num = 10;
    return num;
}
  • ii. Function with return type with arguments:
    This function takes parameters and returns a value. 
    int sum(int a, int b) {
    return a + b;
}
  • iii. Function with no return type with no arguments:
    This function takes no parameters and returns nothing. 
    void display() {
    printf("Hello World");
}
  • iv. Function with no return type with arguments:
    This function takes parameters but returns nothing. 
    void greet(char name[]) {
    printf("Hello, %s", name);
}

3. Storage Classes

Storage classes define the scope, lifetime, and visibility of variables in C/C++ programming.

  • i. Automatic (auto):
    Default storage class for local variables. Declared inside a function and not accessible outside. 
    void test() {
    auto int x = 5;
}
  • ii. External (extern):
    Used to declare a global variable that is defined in another file or outside the current scope. 
    extern int count;
  • iii. Register:
    Suggests that the variable be stored in a CPU register for faster access (not guaranteed). 
    register int speed = 100;
  • iv. Static:
    Retains the value of a variable even after the function exits. Scope remains local, but the lifetime is entire program. 
    void counter() {
    static int count = 0;
    count++;
    printf("%d", count);
}

4. Recursive Function with Syntax and Example

Recursive Function:
A recursive function is a function that calls itself to solve a smaller instance of the same problem. It must have a base condition to stop the recursion, otherwise it leads to infinite calls.

Syntax:

return_type function_name(parameters) {
    if (base_condition)
        return result;
    else
        return function_name(smaller_problem);
}

Example: Factorial using Recursion

int factorial(int n) {
    if (n == 0)
        return 1;
    else
        return n * factorial(n - 1);
}

Function Call: factorial(5) will return 120.

5. Demonstration of Passing Array to a Function

In C/C++, arrays are passed to functions by passing the base address of the array. You can either pass a one-dimensional or multi-dimensional array.

Example: Passing a 1D Array

#include <stdio.h>

void displayArray(int arr[], int size) {
    for (int i = 0; i < size; i++) {
        printf("%d ", arr[i]);
    }
}

int main() {
    int nums[] = {1, 2, 3, 4, 5};
    displayArray(nums, 5);
    return 0;
}

Output: 1 2 3 4 5

Example: Passing a 2D Array

void display2D(int arr[2][3]) {
    for (int i = 0; i < 2; i++) {
        for (int j = 0; j < 3; j++) {
            printf("%d ", arr[i][j]);
        }
        printf("\n");
    }
}

int main() {
    int matrix[2][3] = {{1, 2, 3}, {4, 5, 6}};
    display2D(matrix);
    return 0;
}

Output:
1 2 3
4 5 6
